Общая идея проекта
Проект представляет собой автономный интерактивный веб-инструмент, предназначенный для помощи авторам социальных, культурных, образовательных, туристических, научных, технологических и общественных проектов.
Основная задача системы — помочь пользователю превратить идею в качественно проработанный проект, соответствующий требованиям большинства российских грантовых, конкурсных и экспертных площадок.
Проект не является генератором текста на базе искусственного интеллекта.
Проект является экспертно-методическим цифровым помощником, который пошагово сопровождает пользователя на всех этапах подготовки инициативы и проектной документации.

Предпосылки создания
В ходе анализа были изучены:
* Форум «Сильные идеи для нового времени»;
* требования Агентства стратегических инициатив;
* форма подачи идей СИНВ;
* критерии экспертной оценки СИНВ;
* Президентский фонд культурных инициатив;
* методические материалы ПФКИ;
* чек-листы заявителей;
* примеры успешных проектов;
* формы конкурсных заявок;
* рекомендации экспертов;
* материалы по проектному управлению.
Анализ показал наличие системной проблемы.
Большинство авторов проектов:
* плохо понимают требования экспертов;
* не умеют описывать проблему;
* неверно формулируют цели и задачи;
* не связывают результаты с проблемой;
* не умеют обосновывать значимость проекта;
* не понимают логику оценки заявок.
В результате многие сильные идеи получают слабую проектную упаковку.

Основная цель проекта
Повысить качество подготовки проектов и инициатив за счёт создания интерактивного цифрового помощника, который сопровождает пользователя от идеи до готовой проектной заявки.

Основная задача проекта
Снизить барьер входа для авторов проектов, впервые сталкивающихся с проектной деятельностью, грантовыми конкурсами и экспертной оценкой.

Принцип работы
Основной принцип системы:
«Сначала объясни — потом попроси заполнить»
Перед каждым разделом пользователь получает:
* объяснение раздела;
* критерии оценки;
* примеры;
* типовые ошибки;
* рекомендации;
* чек-лист самопроверки.
Только после этого система предлагает заполнить соответствующее поле.

Отличие от существующих решений
Большинство существующих сервисов работают по схеме:
* форма заявки;
* генерация текста через ИИ;
* автоматическое заполнение полей.
ПроектКонструктор работает по другой логике:
* обучение через действие;
* развитие проектного мышления;
* сопровождение пользователя;
* объяснение логики экспертов;
* подготовка качественного проекта независимо от конкурса.

Система строится не вокруг конкретного конкурса.

Логика построения проекта
Каждый проект независимо от направления должен отвечать на ряд ключевых вопросов:
* В чём проблема?
* Для кого проект?
* Почему он нужен?
* Что предлагается сделать?
* Чем решение лучше существующих?
* Кто будет реализовывать?
* Какие ресурсы нужны?
* Какие результаты будут достигнуты?
* Как будет измеряться успех?
* Что будет после завершения проекта?
Именно вокруг этих вопросов строится структура конструктора.

Уровень 1. Первичная диагностика
Пользователь отвечает на базовые вопросы.
Система определяет:
* стадию зрелости;
* тип проекта;
* уровень готовности.

Уровень зрелости
Варианты:
Идея
Есть понимание проблемы и предполагаемого решения.
Инициатива
Проведены первые исследования и обсуждения.
Проект
Есть команда, план действий и ресурсы.
Практика
Проект реализован и подтвердил эффективность.
Масштабирование
Практика готова к тиражированию.

Уровень 16. Итоговый паспорт проекта
После заполнения всех разделов система автоматически формирует комплект итоговых материалов на основании введённых данных.
Пользователь получает собранный в единую структуру проект без необходимости вручную переносить информацию между документами.

Автоматическое формирование материалов
Система автоматически собирает данные из всех заполненных разделов и формирует:
Паспорт проекта
Краткая структурированная версия проекта.
Включает:
* название;
* тематику;
* стадию зрелости;
* проблему;
* целевую аудиторию;
* цель;
* задачи;
* описание решения;
* мероприятия;
* результаты;

* партнёров;
* ресурсы;

Полное описание проекта
Расширенная версия проекта, объединяющая все заполненные разделы в единый связный документ.

Краткая презентационная справка
Сжатое описание проекта для:

* партнёров;

Черновик конкурсной заявки

Ключевая ценность проекта
Проект не создаёт новые требования.
Проект делает существующие требования понятными.
Вместо подхода:
«Вот форма заявки — разбирайтесь сами»
пользователь получает:
«Вот критерий, вот объяснение, вот пример, вот ошибки, вот рекомендации, вот поле для заполнения»

Итоговая миссия проекта
Помочь любому человеку — независимо от опыта, профессии и уровня подготовки — превратить идею в качественно проработанный проект, понятный экспертам, партнёрам, грантодателям и органам власти.
